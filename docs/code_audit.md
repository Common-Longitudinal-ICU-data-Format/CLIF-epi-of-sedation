# Peer Review Audit: Epidemiology of Sedation in Mechanical Ventilation

**Date:** 2026-03-31

**Scope:** All scripts in `code/` (01_cohort through 07_analysis, _utils.py), cross-referenced against `docs/analysis_plan.md`, `docs/Sedative Exposure Plan.md`, `docs/intub_extub_specs.md`, and the CLIF mCIDE specification.

**Reviewer assessment:** The analysis pipeline is well-structured, clearly documented, and demonstrates strong engineering practices (modular marimo notebooks, CLIF standard compliance, intermediate parquet caching). The cohort, exposure, and outcome definitions generally align with prior literature (Mehta et al., Seymour et al.). However, several issues ranging from potential logic bugs to methodological concerns warrant attention before journal submission.

Findings are organized by severity. Each includes the file, line(s), and a concrete recommendation.

## CRITICAL: Likely Bugs Affecting Correctness

### C1. T-piece regex matches literal "1" instead of clinical "T-piece"

- **File:** `03_outcomes.py:186`

- **Code:** `regexp_matches(device_name, 't1[\\s_-]?piece')`

- **Issue:** The regex requires a literal digit `1` after `t`, so it matches "t1-piece", "t1 piece", etc. The standard clinical device is called "T-piece" (no digit). This pattern would fail to identify actual T-piece trials unless a specific EHR vendor happens to use "T1-piece" naming. Note: `docs/analysis_plan.md:185` records this same regex as the spec, so the bug originates in the specification.

- **Impact:** SBT events detected via T-piece would be missed, underestimating SBT completion rates.

- **Recommendation:** Change to `t[\s_-]?piece` (remove the `1`). Validate against actual `device_name` values from at least one site. Update both code and analysis_plan.md.

### C2. SBT mode_category case mismatch with CLIF mCIDE

- **File:** `03_outcomes.py:184`

- **Code:** `mode_category IN ('pressure support/cpap')`

- **CLIF mCIDE value:** `Pressure Support/CPAP` (title case)

- **Issue:** DuckDB string comparison is case-sensitive by default. If the data preserves mCIDE casing, this predicate matches nothing.

- **Impact:** Primary SBT detection pathway would fail entirely, producing all-zero `sbt_done` outcomes.

- **Recommendation:** Use `LOWER(mode_category) IN ('pressure support/cpap')` or verify that clifpy's waterfall processing lowercases mode_category. Add an assertion checking that at least some SBT states are detected.

### C3. Column name mismatch between 03_outcomes and 05_analytical_dataset

- **Files:** `03_outcomes.py:409` produces `_success_extub`; `05_analytical_dataset.py:120` references `o.success_extub` (no leading underscore). Also `o.n_hrs` on line 118 -- no such column exists in the daily aggregation output.

- **Impact:** The pipeline would fail at runtime if 03 is re-run with current code, then 05 is run. Currently works only because the parquet file was generated by a prior code version.

- **Recommendation:** Reconcile column names. In 03, rename to `success_extub` (without underscore) for consistency, or update the reference in 05. Add or remove `n_hrs` as appropriate.

### C4. Vasopressor ASOF join has no staleness window

- **File:** `04_covariates.py:360-405`

- **Issue:** Each vasopressor is ASOF-joined to shift-change grids with `m.admin_dttm <= g.event_dttm` but no upper bound on how old the record can be. A vasopressor stopped days ago will be carried forward to the current shift change. Unlike pH (which has a 12-hour check, line 153), vasopressors have no staleness filter.

- **Impact:** Overestimates vasopressor support and NEE for patients whose pressors were discontinued.

- **Recommendation:** Add a staleness check (e.g., within 4-6 hours) or implement explicit forward-fill-then-zero logic using the medication stop events. Alternatively, compute a true point-in-time rate using the forward-fill from `cont_sed_dose_by_hr`-style logic.

### C5. P/F ratio ASOF join has no staleness window

- **File:** `04_covariates.py:206-233`

- **Issue:** The PO2 lab ASOF join has no time constraint. A PO2 drawn 3 days ago could be paired with a current FiO2, producing a clinically meaningless P/F ratio. The pH covariate correctly implements a 12-hour staleness check, but P/F does not.

- **Impact:** Stale P/F values may misclassify respiratory severity. This asymmetry with pH handling is inconsistent.

- **Recommendation:** Apply the same 12-hour (or similar) staleness window used for pH. Mark stale P/F as 'missing'.

## HIGH: Methodological Concerns

### H1. Continuous dose forward-fill may persist past infusion stop

- **File:** `02_exposure.py:240-252`

- **Code:** `LAST_VALUE(COLUMNS('_cont') IGNORE NULLS) OVER (PARTITION BY hospitalization_id ORDER BY event_dttm)`

- **Issue:** Forward-fills the last observed infusion rate indefinitely. If a sedation drip is stopped but the stop event records a non-zero dose (varies by EHR), that rate continues to accrue across all subsequent hours until another event occurs.

- **Impact:** Could systematically overestimate sedation exposure, especially for drugs stopped without an explicit rate-to-zero event.

- **Recommendation:** Validate that stop events in the CLIF data produce `med_dose = 0`. Consider adding a maximum carry-forward window (e.g., 4 hours) to limit stale rate propagation.

### H2. FULL JOIN creates rows outside the observation window

- **File:** `02_exposure.py:212-225`

- **Code:** `FULL JOIN cont_sed_w m USING (hospitalization_id, event_dttm)`

- **Issue:** A FULL JOIN introduces medication events from OUTSIDE the hourly grid timeframe (before IMV start or after IMV end). These out-of-window rows participate in the forward-fill and duration calculations.

- **Impact:** Pre-IMV sedation events could contaminate the first hours of the observation window.

- **Recommendation:** Change to `LEFT JOIN` (grid as left table) to ensure only in-window events are included. Out-of-window medication events would join via the forward-fill mechanism anyway.

### H3. NMB duration logic conflates administration interval with infusion duration

- **File:** `01_cohort.py:368-414`

- **Issue:** Duration is computed as `LEAD(admin_dttm) - admin_dttm`, which measures the TIME BETWEEN consecutive NMB administration records, not the actual paralysis duration. If two boluses are 6 hours apart, this counts 360 minutes of NMB, even though the pharmacological effect may be 30-60 minutes per bolus. The threshold of >60 minutes is meant to exclude meaningfully paralyzed patients, but this calculation overestimates NMB duration.

- **Impact:** Over-exclusion of patient-days, potentially introducing selection bias.

- **Recommendation:** For continuous NMB infusions, this logic is reasonable. For intermittent boluses (which cisatracurium/rocuronium often are), consider using a fixed pharmacological duration per administration (e.g., 60 min per bolus) rather than inter-dose interval.

### H4. SBT detection: NULL PEEP/PS values silently exclude valid SBTs

- **File:** `03_outcomes.py:183-187`

- **Issue:** `peep_set <= 8 AND pressure_support_set <= 8` evaluates to NULL (not TRUE) when either value is missing. After the waterfall forward-fill, some values may still be NULL (start of hospitalization, device changes). These rows are not counted as SBT even if the patient is on PS/CPAP mode.

- **Impact:** Underestimates SBT rates in patients with sparse respiratory charting.

- **Recommendation:** Use `COALESCE(peep_set, 0) <= 8 AND COALESCE(pressure_support_set, 0) <= 8`, or add an explicit NULL handling strategy with clinical justification.

### H5. Failed extubation check doesn't exclude tracheostomy transitions

- **File:** `03_outcomes.py:235-242`

- **Issue:** The `_intub` flag in `sbt_t1` doesn't check `tracheostomy = 0` (unlike the specification in `docs/intub_extub_specs.md`). The `_fail_extub` subquery scans `sbt_t1` including tracheostomy rows. A patient extubated and then tracheostomized within 24h (a planned surgical pathway) would be misclassified as "failed extubation."

- **Impact:** Inflates failed extubation rate; correspondingly deflates successful extubation rate.

- **Recommendation:** Add `AND sbt_t1.tracheostomy = 0` to the EXISTS subquery, consistent with the specification document.

### H6. Table 1 describes Day 1 only, but models use all patient-days

- **File:** `06_table1.py:96-103`, `07_analysis.py:231-274`

- **Issue:** Table 1 filters to `_nth_day = 1`, but the GEE and logistic models use all patient-days (days 1 through N). The descriptive statistics don't characterize the actual analytic population.

- **Impact:** Readers cannot assess whether the modeled population's covariate distributions match what Table 1 shows. Violates STROBE recommendation that Table 1 describe the study population.

- **Recommendation:** Add a Table 1 for the full analytic population (all patient-days), or at minimum report summary statistics (median days contributed per patient, distribution of `_nth_day`).

### H7. T-test performed on hourly data violates independence

- **File:** `07_analysis.py:104-121`

- **Issue:** The day vs night t-test is performed on `sed_dose_by_hr` (hourly level), not patient-day aggregates. Multiple hours from the same patient on the same day are treated as independent observations, massively inflating the effective sample size and producing artificially small p-values.

- **Impact:** Type I error inflation; reported p-values are not valid.

- **Recommendation:** Aggregate to patient-day level before the t-test, or use a mixed model / paired test that accounts for within-patient clustering.

## MEDIUM: Design and Operationalization Concerns

### M1. Partial-day bias in dose measurement

- **Files:** `05_analytical_dataset.py:158` (excludes day 0), `_utils.py:23` (day boundary at 7am)

- **Issue:** Day 0 is excluded, but Day 1 may also be a partial day. If intubation occurs at 3pm, the "day" shift of Day 1 has only 4 hours of observation (3pm-7pm) while the night shift has 12 hours. The dose "diff" (night - day) would be biased toward positive values simply due to unequal observation time.

- **Recommendation:** Either normalize doses by hours observed, or exclude patient-days with <X hours of observation in either shift. At minimum, report the distribution of hours per shift for Day 1.

### M2. COALESCE(dose, 0) conflates "no drug given" with "period not observed"

- **File:** `05_analytical_dataset.py:123-128`

- **Issue:** Missing dose values (e.g., when a partial day has no night shift) are filled with 0. A patient intubated at 10pm on Day 1 would have `prop_day = 0` not because no propofol was given during the day, but because the day shift wasn't observed. The dose diff would incorrectly show a "nighttime increase."

- **Recommendation:** Distinguish between true zeros (drug not administered) and structural missingness (period not yet started). Consider excluding partial-day observations from the regression.

### M3. Missing key covariates risk residual confounding

- **Files:** `05_analytical_dataset.py:133`, `docs/analysis_plan.md:148-158`

- **Issue:** Only age is included as a demographic covariate. Sex, race/ethnicity, illness severity (SOFA), comorbidity burden (Charlson), sedation depth (RASS), and ICU type are all listed in the analysis plan as "FUTURE." Without these, the regression models have substantial residual confounding.

- **Impact:** Estimated associations between nighttime sedation dose changes and outcomes are likely biased.

- **Recommendation:** Implement at minimum SOFA score (available via `clifpy.calculate_sofa2_daily()`) and sex before submission. Discuss unmeasured confounding in limitations.

### M4. Inconsistent modeling strategy across outcomes

- **File:** `07_analysis.py:230-274`

- **Issue:** SBT done uses GEE with binomial family. Successful extubation uses logistic regression with clustered standard errors. Both are valid, but the inconsistency is unexplained. GEE estimates population-averaged effects; logistic regression with clustered SEs estimates subject-specific effects (adjusted for clustering). The two are not directly comparable.

- **Recommendation:** Use the same modeling approach for both outcomes and justify the choice. GEE with exchangeable correlation is more standard for both outcomes in this setting.

### M5. GEE working correlation structure unspecified

- **File:** `07_analysis.py:238`

- **Issue:** The `statsmodels.gee()` call doesn't specify a `cov_struct`, defaulting to independence. For longitudinal patient-day data with potentially autocorrelated outcomes, independence is a strong (likely wrong) assumption that affects efficiency and may affect inference.

- **Recommendation:** Test exchangeable and AR(1) structures. Report the QIC for model comparison.

### M6. Categorical covariate reference levels uncontrolled

- **File:** `07_analysis.py:234-236, 261-263`

- **Issue:** pH and P/F categorical levels include 'missing' as a category. statsmodels picks the alphabetical first level as reference. For `ph_level`, this could be `missing` or `ph_72_73` depending on sort order. The reference level affects interpretation.

- **Recommendation:** Explicitly set reference levels via `C(ph_level_7am, Treatment(reference='ph_73_74'))` in the formula, using a clinically meaningful normal range as reference.

### M7. Venous pH adjustment (+0.05) lacks citation and may be inaccurate

- **File:** `04_covariates.py:151`

- **Code:** `COALESCE(ph_arterial, ph_venous + 0.05)`

- **Issue:** The +0.05 offset is a rough population average. In critically ill patients, the arteriovenous pH gap can range from 0.01 to 0.10+. No citation is provided.

- **Recommendation:** Cite the source (e.g., Brandenburg & Dire, Ann Emerg Med 1998). Consider sensitivity analysis with and without venous pH substitution.

### M8. Encounter stitching computed but not wired in

- **File:** `01_cohort.py:106-111`

- **Issue:** Patients readmitted within 12 hours may have their IMV streak artificially split across hospitalizations. The stitching computation exists but isn't used.

- **Recommendation:** Either connect it to the downstream pipeline or remove the dead code. Document the decision in the methods section.

### M9. `_end_mode` computed identically to `_start_mode`

- **File:** `03_outcomes.py:264-265`

- **Code:** Both use `ANY_VALUE(mode_category ORDER BY recorded_dttm)` -- same sort order

- **Issue:** Both return the earliest mode in the block. `_end_mode` should use `ORDER BY recorded_dttm DESC`.

- **Impact:** Low -- these columns don't appear to be used in downstream analysis.

- **Recommendation:** Fix the sort order or remove the unused column.

## LOW: Documentation, Style, and Minor Issues

### L1. CONSORT flow incomplete

- **File:** `01_cohort.py:441-464`

- **Issue:** CONSORT JSON has 3 steps (ICU stays -> IMV >=24h -> NMB exclusion). Missing: null-age exclusion (05:144), day-0 exclusion (05:158), null-next-day-outcome exclusion (05:158). The reader cannot trace from initial N to final analytic N.

- **Recommendation:** Add the downstream exclusion steps to the CONSORT, or generate a second CONSORT in 05_analytical_dataset.py.

### L2. Typo: `_withdrawl_lst` throughout

- **Files:** `03_outcomes.py`, `05_analytical_dataset.py`

- **Issue:** Should be `_withdrawal_lst`. Present in ~10 locations.

- **Recommendation:** Rename for clarity before submission.

### L3. Dexmedetomidine and ketamine excluded without justification

- **File:** `docs/analysis_plan.md:73-74`

- **Issue:** These are common ICU sedatives. Their exclusion from equivalency calculations should be justified in the manuscript (e.g., different mechanism of action, inability to convert to equivalency units).

### L4. Morphine not included in fentanyl equivalency

- **File:** `docs/analysis_plan.md:75, 116`

- **Issue:** The analysis plan acknowledges this gap: "10mg morphine = 100mcg fentanyl" conversion exists but morphine is not loaded. Some patients may receive morphine infusions.

- **Recommendation:** Either add morphine or report the proportion of patients receiving it to quantify the unmeasured exposure.

### L5. No sensitivity analyses on algorithmic thresholds

- **Thresholds:** 24h IMV (cohort), 60min NMB (exclusion), 30min SBT (outcome), 12h pH (staleness)

- **Recommendation:** Reviewers will likely request at least one sensitivity analysis (e.g., SBT >= 20min or >= 45min).

### L6. Hourly grid extends one hour past IMV streak end

- **File:** `01_cohort.py:310`

- **Code:** `_end_hr: date_trunc('hour', _end_dttm) + INTERVAL '1 hour'`

- **Issue:** Adds one extra hour beyond streak end. The forward-fill then carries the last sedation rate into this extra hour.

- **Impact:** Minimal -- one extra hour of data per patient.

- **Recommendation:** Remove the `+ INTERVAL '1 hour'` or document why it's needed.

### Overall concordance assessment

The `analysis_plan.md` is an honest self-audit -- most plan-vs-code gaps are already documented there with accurate statuses. The primary concerns for a peer reviewer are:

1. **Scope of unimplemented covariates is large** -- 9 of 13 planned covariates are FUTURE. For a journal submission, this is a significant limitation that must be prominently discussed.

2. **The fundamental analysis design changed** from the original plan (hour 24/72 snapshot) to the current approach (daily aggregation with next-day LEAD outcomes) without documentation of when/why this decision was made.

3. **Intermittent/bolus inclusion** contradicts the original plan's "continuous drip only" specification. This changes the exposure definition and should be explicitly justified.

4. **RASS is the single most impactful missing covariate** -- it directly captures sedation depth, which is the mechanistic pathway through which dose changes affect SBT and extubation outcomes. Without it, the models cannot distinguish "higher dose -> deeper sedation -> worse outcomes" from "higher dose -> same sedation depth -> no effect."
